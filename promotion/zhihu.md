# 知乎文案

## 文章一：问题讨论风

### 标题

复试线低就代表好考吗？

### 正文

不一定。

复试线只是进入复试的门槛，不等于最终录取难度。很多考研择校误判，就是从把“复试线”当成“录取线”开始的。

真正需要同时看的，至少包括：

- 近三年国家线和专业复试线；
- 招生计划是否包含推免和专项；
- 实际留给统考的名额；
- 复试名单和拟录取名单；
- 拟录取最低分、中位数和均分；
- 复试权重和是否保护一志愿；
- 专业课是否变化，比如改考 408。

举个抽象例子：某专业复试线只有 280，看起来很友好。但如果拟录取中位数是 350，最低录取分也明显高于复试线，那它就不是“低分好上岸”，而是“低门槛进复试，高分才能录取”。

另一个常见误区是招生人数。

招生目录写 60 人，不代表统考招 60 人。计划数可能包含推免、专项、非全，甚至是学院整体口径。对统考考生来说，更关键的是扣掉推免后还剩多少名额。

所以我做了一个开源项目：`kaoyan-navigator-skill`。

它不是一个“预测神器”，而是给 AI Agent 用的择校调研流程。输入学校、学院和专业后，Agent 会先去找学校研究生院、招生网、学院官网、研招网、复试细则和拟录取名单，再整理数据和来源，最后输出带置信度的建议。

我给它设置了一个很重要的规则：

> 搜不到的数据，不允许自动脑补。

如果拟录取名单没有公开，它就应该告诉用户缺少这项材料，并说明这会影响“复试线是否虚低”的判断，而不是为了报告完整随便填一个数字。

这个项目目前支持 Codex、Claude Code 和其他可读取 Markdown Skill 的 Agent，也可以导入 CSV/JSON 数据。

项目地址：

```text
https://github.com/mcxiaoxiao/kaoyan-navigator-skill
```

我更希望它被当作一套公开的择校研究流程，而不是制造焦虑的工具。考研择校最终还是个人决策，但至少我们可以让 AI 先把证据找齐，把不确定性说清楚。

适合话题：`考研择校`、`考研信息差`、`计算机考研`、`AI Agent`、`开源项目`。

## 文章二：方法论风

### 标题

怎样判断一个考研目标院校是不是被低估了难度？

### 正文

判断一个考研目标院校难不难，不能只看单个数字。

我会把它拆成四类问题。

第一，分数是不是有欺骗性？

复试线低，不代表录取分低。真正需要看拟录取名单里的最低分、中位数和均分。如果复试线贴着国家线，但实际录取中位数高很多，说明复试线只是低门槛，不是低难度。

第二，名额是不是真的多？

招生计划需要拆成计划总数、推免、专项、非全和统考名额。对大多数统考生来说，“统考名额”比“招生计划”重要得多。

第三，趋势是不是在变差？

如果连续缩招、推免比例上升、报考热度上升，哪怕去年分数线不高，今年也不能简单判断为安全。

第四，数据是不是同一个口径？

同一个学校的不同学院、不同专业代码、全日制和非全日制不能混用。很多网上报录比和录取分数，问题就出在口径不清。

基于这套思路，我把择校分析做成了一个 AI Agent Skill：`kaoyan-navigator-skill`。

它的目标不是替你决定学校，而是把“查数据、核口径、找证据、说明不确定性”这套流程固定下来。

它会要求 Agent 先做这些事：

1. 确认学校、学院、专业代码和培养方式；
2. 查近三年复试线和国家线；
3. 查招生计划、推免和统考名额；
4. 查复试名单、拟录取名单和录取分布；
5. 记录每个数字的来源、年份和口径；
6. 对缩招、推免、复试线虚低、改考等信号做分项判断；
7. 对每个高风险结论找反方证据。

最重要的是：数据不足不等于低难度。

如果某个学校不公开拟录取名单，那么这项分析的置信度就应该降低，而不是自动认为没有问题。

项目地址：

```text
https://github.com/mcxiaoxiao/kaoyan-navigator-skill
```

如果你正在择校，可以把它当作一份检查清单；如果你在做 Agent 或教育数据工具，也可以把它当作一个可复用 Skill 的案例。

## 文章三：开源故事风

### 标题

我为什么把考研择校做成一个开源 AI Skill？

### 正文

因为考研择校里有太多“看似有数据，实际没看懂口径”的情况。

有些经验帖会说某学校好考，理由是复试线不高；有些推荐会说某专业招生很多，却没扣推免；有些报录比看起来很友好，但其实是学院口径，不是专业口径。

这些问题不完全是人的问题，AI 也很容易犯。

如果你直接问一个通用 AI：“某某大学某专业稳不稳？”它可能会给你一段看起来很完整的分析。但问题是，它有没有查今年招生目录？有没有确认专业代码？有没有看拟录取名单？有没有区分全日制和非全日制？有没有说明哪些数据找不到？

所以我做了 `kaoyan-navigator-skill`。

它不是为了让 AI 变成“择校大师”，而是给 AI 加一套工作纪律：

- 先搜索官方来源；
- 再整理数据口径；
- 找不到就说找不到；
- 每个判断都写证据；
- 高风险结论要找反证；
- 最后输出建议，但不承诺结果。

这个项目的名字叫 navigator，是因为我不希望它给人一种“判你有风险”的感觉。它更像导航：帮你看路况、提示拥堵和分岔，但方向盘还是在你手里。

目前仓库里有 Agent 使用的 `SKILL.md`、数据搜索手册、风险与置信度模型、报告模板、CSV/JSON schema、示例数据和校验脚本。

项目地址：

```text
https://github.com/mcxiaoxiao/kaoyan-navigator-skill
```

如果你有某个专业的公开数据、拟录取名单解析经验，或者觉得某个指标设计不合理，欢迎来提 Issue。一个好的择校工具，应该允许被质疑、被纠错、被补充。
