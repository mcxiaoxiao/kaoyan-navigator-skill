# 贡献指南

感谢你帮助改进 `kaoyan-navigator-skill`。

## 可以贡献什么

- 修正错误的招生数据口径或风险阈值；
- 增加院校官网、研招系统和 PDF 的搜索策略；
- 改进 CSV/JSON 校验、OCR 或名单统计脚本；
- 增加 Codex、Claude Code 或其他 Agent 的兼容性；
- 提供经过脱敏的真实使用案例；
- 改进中文文档、示例和测试。

## 提交 Issue

提交前请搜索已有 Issue。报告数据问题时请尽量提供：

- 学校、学院、专业代码、培养方式和年份；
- 官方来源 URL 或文件；
- 当前结果与预期结果；
- 数据口径差异，例如计划数是否包含推免、非全或专项。

请勿公开个人姓名、准考证号、手机号等敏感信息。

## 提交 Pull Request

1. Fork 仓库并创建独立分支。
2. 保持变更聚焦，避免混入无关格式化。
3. 修改风险规则时说明依据和可能影响。
4. 修改脚本时运行：

```bash
python3 -m py_compile scripts/validate_data.py
python3 scripts/validate_data.py examples/sample.csv
```

5. 更新相关文档和示例。
6. 填写 PR 模板中的检查项。

## 数据与证据规范

- 优先使用研招网、学校研究生院、招生网和学院官网。
- 第三方信息应尽量回溯到官方原文。
- 社交媒体只用于热度和体验信号，不直接证明招生事实。
- 不要用 `0` 表示未知数据。
- 不得提交未经脱敏的考生个人信息。
- 不得伪造数据、来源、预测准确率或录取概率。

## 开发原则

- `SKILL.md` 保持精炼，详细规则放入 `references/`。
- 可重复、格式敏感的操作优先封装到 `scripts/`。
- 新增引用文件时，从 `SKILL.md` 直接链接，避免多层跳转。
- 保持 Python 脚本兼容 Python 3.9 及以上版本，除非变更有明确理由。
