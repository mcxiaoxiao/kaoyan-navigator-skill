# 历年数据获取流程

这个流程专门解决“先自己找，找不到再问用户”的问题。

## 三种输入模式

1. **用户已有 CSV / JSON**
   - 先跑 `scripts/validate_data.py`
   - 再统一字段名、年份和研究对象
   - 直接进入分析

2. **用户只有 PDF / 网页 / 截图**
   - 先抽取文本或表格
   - 把每一年的数据整理成一行
   - 再跑校验脚本

3. **用户什么都没有**
   - 先按公开来源顺序全网检索
   - 找到一部分就先结构化
   - 关键字段仍缺失时，再问用户最小必要材料

## 公开检索顺序

按这个顺序找：

1. 学校研究生院、招生网、学院官网
2. 官方 PDF、招生简章、复试细则、拟录取公示
3. 学校官方公众号、政府媒体、可复核名单统计
4. 教育媒体、考研机构、聚合站
5. 知乎、小红书、B 站、论坛，仅作弱信号

## 具体执行逻辑

1. 先规范化研究对象：`学校 / 学院 / 专业代码 / 专业名称 / 培养方式 / 入学年份`
2. 再定义时间窗：`Y-3` 到 `Y-1`
3. 用 `scripts/research_plan.py` 生成查询模板和缺失字段清单
4. 逐年搜集并记录来源等级
5. 任何冲突都保留，不强行合并
6. 只有在公开检索失败后，才向用户要补充材料

## 最小补充追问模板

- 我已经找到 `2024-2026` 年的招生目录，但没找到该专业的拟录取名单，因此还不能判断“复试线虚低”。  
  请上传任一年拟录取名单 PDF / 截图，或给我分数列表。

- 我已经找到招生计划，但没法确认统考名额是否被推免挤压。  
  请补充该年的推免公示、学院说明或专业目录脚注。

## 归档原则

- CSV / JSON 一行只表示一个“年份 + 同口径专业”
- `0` 只表示官方明确为零，未知请留空
- 同名专业但学院不同、代码不同、培养方式不同，不得混用
- 不能为补表而猜数
