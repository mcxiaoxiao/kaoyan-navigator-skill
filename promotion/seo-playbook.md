# SEO 优化手册

这份手册用于优化 CSDN、知乎、小红书三类推广稿。原则是先有人味，再做关键词；不要为了排名把文章写成关键词列表。

## 参考原则

### 通用 SEO

- 内容要先服务真实读者，而不是为了搜索排名硬写。优先提供原创经验、完整解释和可验证信息。
- 标题和描述要准确概括页面内容，不要夸大承诺。
- 图片放在相关段落附近，并写清楚图片展示了什么。
- 不要为了所谓字数或关键词密度写废话。

### CSDN

- 读者多是开发者，标题可以放技术词：`Codex Skill`、`AI Agent`、`数据 schema`、`PDF 解析`。
- 开头 100 字内出现主关键词和项目名。
- 用小标题、代码块、目录感结构提升可读性。
- 结尾给 GitHub 仓库、运行命令和可贡献方向。

### 知乎

- 用问题型标题承接搜索意图，例如“复试线低就代表好考吗？”。
- 首段先回答问题，再展开方法论。
- 少用营销语，多用“我为什么这么做”“我踩到什么坑”“这个判断有什么边界”。
- 话题建议控制在 3 到 5 个，避免堆砌。

### 小红书

- 标题短，核心词前置，例如“考研择校”“复试线”“招生人数”。
- 正文前 100 字内自然出现主关键词。
- 每篇只打一个强钩子：复试线、统考名额、AI 导航，别同时讲太多。
- 标签组合使用泛词、精准词和长尾词。
- 引导评论时问具体问题，不要只写“欢迎交流”。

## 关键词矩阵

### 核心词

```text
考研择校
考研信息差
复试线
统考名额
推免比例
拟录取名单
冲稳保
```

### 长尾词

```text
复试线低就代表好考吗
招生人数和统考名额区别
考研拟录取名单怎么看
考研择校 AI 工具
Codex Skill 怎么写
AI Agent 考研择校
408 考研择校避坑
```

### 项目词

```text
kaoyan-navigator-skill
```

## 三平台标题公式

### CSDN

```text
[技术词] + [场景] + [项目动作]
Codex Skill 实战：把考研择校数据调研做成可复用 Agent 流程
```

### 知乎

```text
[用户问题] + [反常识解释]
复试线低就代表好考吗？我做了一个开源 AI Skill 后更不敢只看这一个数
```

### 小红书

```text
[核心词前置] + [强钩子]
考研择校别只看复试线
招生50人≠统考50人
```

## 图片建议

首图文件：

```text
promotion/assets/github-repo-home.png
```

推荐 alt：

```text
GitHub 开源仓库 mcxiaoxiao/kaoyan-navigator-skill 首页截图，展示考研择校导航 Skill 的 README、promotion 目录和项目 topics。
```

推荐封面字：

```text
考研择校别只看复试线
招生50人≠统考50人
AI先查证据再给建议
```

## 避免事项

- 不写“百分百预测”“稳上”“必爆”。
- 不用过密关键词破坏阅读。
- 不公开未脱敏的拟录取名单个人信息。
- 不把社媒推荐当官方事实。
- 不把“找不到数据”包装成“低难度”。
