# 推广素材

## 项目定位

`kaoyan-risk-skill` 是一个面向 Codex、Claude Code 和通用 AI Agent 的考研择校调研 Skill。核心差异是“先搜索和核验公开数据，再给风险结论”，并明确展示证据等级、缺失数据和置信度。

## 核心卖点

- 自动调研复试线、招生计划、统考名额、推免和拟录取数据；
- 检测缩招、爆热、复试线虚低、改考 408 等风险；
- 公开数据找不到后才请用户补充；
- 支持 CSV、JSON、PDF、截图和网页材料；
- 适配 Codex、Claude Code 和其他可读取 Markdown Skill 的 Agent；
- 所有结论强调来源和不确定性，不承诺“稳上”。

## 发布标题备选

1. 开源一个会自己查官网数据的考研择校 AI Skill
2. 别只看复试线：用 AI Agent 审计考研统考名额和录取风险
3. Kaoyan Risk Skill：让 Codex 帮你做有证据的考研择校分析
4. 从“预测会不会爆”到多 Agent 风险研究：一个通用考研 Skill

## 首发文案

最近把考研择校分析整理成了一个开源 Skill：`kaoyan-risk-skill`。

它不会直接凭印象说某校“稳”或“必爆”，而是先寻找学校官网、研究生院、招生目录、复试细则和拟录取名单，整理近三年的复试线、统考名额、推免比例、竞争和录取分布，再评估缩招、热度、复试线虚低、科目变化等风险。

如果公开资料找不到，它会明确指出缺少哪项数据、影响什么结论，并只请求用户补充最小必要材料。项目支持 Codex、Claude Code 和其他 Agent，也能导入 CSV/JSON 数据。

欢迎提交真实案例、搜索策略和指标改进。招生数据请尽量附官方来源，并注意考生隐私。

## 推广注意事项

- 不使用“100% 准确”“稳上”“必爆”等承诺；
- 不展示未经脱敏的拟录取个人信息；
- 不把社交平台热度包装成官方招生事实；
- 发布案例时写明数据年份和访问日期；
- 引导用户 Star、提交 Issue 和贡献数据口径，不做强制引流。
