## 文章一：讨论风

### 标题

复试线低就代表好考吗？考研择校别只看这一个数

### SEO 摘要

考研择校不能只看复试线，还要看统考名额、推免比例、拟录取最低分和中位数。本文用开源项目 `kaoyan-navigator-skill` 解释如何让 AI 先查证据再给建议。

### 关键词

```text
考研择校, 复试线, 统考名额, 拟录取名单, 考研信息差, AI Agent
```

### 正文

不一定。

这可能是考研择校里最常见的误会之一。

复试线只是“能不能进复试”的门槛，不是“最终录取难度”。有些专业复试线看起来很低，但拟录取名单里大家的分数并不低。换句话说，你可以低分进复试，但不一定能低分录取。

我之前整理考研数据时，经常碰到这种情况：

- 复试线贴着国家线；
- 招生计划看起来不少；
- 网上还有人说“保护一志愿”“不歧视本科”；
- 但拟录取最低分和中位数一看，并没有那么轻松。

所以我后来做了一个开源小项目：

```text
mcxiaoxiao/kaoyan-navigator-skill
```

旧链接是 `mcxiaoxiao/kaoyan-risk-skill`，现在会跳转到新仓库。名字我也改了，`risk` 对考研来说有点不吉利，还是 `navigator` 更像一个导航工具。

仓库截图：

```markdown
![仓库首页截图](promotion/assets/github-repo-home.png)
```

这个项目不是让 AI 直接预测“会不会爆”，而是让 AI 先做资料整理：

- 查学校官网、研究生院、学院公告；
- 看近三年复试线和国家线；
- 拆招生计划、推免和统考名额；
- 找复试名单和拟录取名单；
- 算录取最低分、中位数、均分；
- 把数据来源和缺失项写清楚。

我给它写了一条很朴素的规则：

> 搜不到的数据，不许自动脑补。

比如拟录取名单没有公开，那就应该说“这一项缺数据”，而不是假装知道录取均分。数据不足，不等于这个学校就安全。

我觉得 AI 做择校，最有价值的地方不是给你一句“冲”或“不冲”，而是帮你把那些分散在 PDF、官网通知和拟录取名单里的信息整理出来。它可以当研究助理，但不能替你做人生决定。

项目地址：

```text
https://github.com/mcxiaoxiao/kaoyan-navigator-skill
```

如果你正在择校，可以把它当检查清单用；如果你做 Agent 或教育数据，也可以把它当一个 Skill 结构参考。

## 文章二：方法论风

### 标题

考研目标院校值不值得冲？我会先查这4类数据

### SEO 摘要

判断考研目标院校是否值得冲，不能只看分数线。本文从复试线、统考名额、近三年趋势和数据口径四个角度，介绍一个开源 AI Skill 的择校检查流程。

### 关键词

```text
考研目标院校, 冲稳保, 复试线虚低, 推免比例, 408考研, kaoyan-navigator-skill
```

### 正文

以前看考研择校，我也会先看复试线。

后来发现不够。

现在如果要判断一个目标院校值不值得冲，我会先看这几件事：

第一，复试线和录取分数差多少。

复试线低不一定友好。如果拟录取中位数高很多，说明复试线只是低门槛，不代表低难度。

第二，招生计划里到底有多少统考名额。

招生 50 人和统考 50 人不是一回事。推免、专项、非全、学院口径，都可能影响这个数字。

第三，近三年趋势是不是变差。

连续缩招、推免上升、改考 408、突然被大量推荐，这些都不一定单独致命，但叠在一起就要小心。

第四，数据是不是同一个对象。

同一个学校，不同学院的数据不能混；同一个专业名，不同专业代码也不能混；全日制和非全更不能混。

我把这套检查流程整理成了一个开源 Skill：

```text
mcxiaoxiao/kaoyan-navigator-skill
```

仓库截图可以放这里：

```markdown
![kaoyan-navigator-skill 仓库截图](promotion/assets/github-repo-home.png)
```

它之前叫 `kaoyan-risk-skill`，如果你点旧链接也能跳过去。我后来改名了，因为这个项目的定位不是“吓唬你有风险”，而是帮你导航：哪里拥堵、哪里看不清、哪里需要补数据。

它会要求 Agent 先确认研究对象，再去找资料。找不到关键数据时，不是继续编，而是明确告诉用户需要补什么，比如：

- 拟录取名单 PDF；
- 复试名单截图；
- 招生目录链接；
- 用户自己整理的 CSV/JSON。

这个小项目让我越来越觉得，AI 在择校场景里最该做的不是“判断”，而是“把判断依据摆出来”。只要依据清楚，用户自己其实会做出更稳的选择。

项目地址：

```text
https://github.com/mcxiaoxiao/kaoyan-navigator-skill
```

也欢迎提 Issue，尤其是数据口径和真实案例。考研择校这种东西，靠一个人的经验很容易偏，开源出来就是希望它能被纠错。

## 文章三：开发者自述风

### 标题

AI Agent 能做考研择校吗？我做了一个开源 Skill 后的想法

### SEO 摘要

AI Agent 可以辅助考研择校，但前提是先搜索官方来源、统一数据口径、记录缺失项。本文介绍 `kaoyan-navigator-skill` 的设计动机和边界。

### 关键词

```text
AI Agent, 考研择校, 开源 Skill, Codex Skill, Claude Code, 教育数据
```

### 正文

起因其实挺小。

我看到很多择校分析都在讲“某校会不会爆”“某专业是不是洼地”，但越看越发现，很多结论的底层数据并不稳。

有的是只看复试线，没看拟录取分数。

有的是只看招生计划，没扣推免。

有的是拿学院口径的报录比，去解释某个专业。

还有的是把社交平台上的“闭眼冲”“性价比高”当成事实。

这些问题，人会犯，AI 也会犯，而且 AI 犯起来更隐蔽，因为它能把一段不太扎实的结论写得很顺。

所以我做了一个 Skill：

```text
mcxiaoxiao/kaoyan-navigator-skill
```

仓库截图：

```markdown
![GitHub 仓库截图](promotion/assets/github-repo-home.png)
```

旧名字是 `kaoyan-risk-skill`。后来我改掉了，原因很简单：risk 太冷了，也有点不吉利。`navigator` 更接近我想要的感觉：它不替你开车，但会告诉你前面哪里堵、哪里需要看路牌。

这个 Skill 主要写给 Codex、Claude Code 这类 Agent。它规定 Agent 做考研择校时应该先：

- 搜官方来源；
- 统一学校、学院、专业代码和培养方式；
- 建立三年数据表；
- 把复试线和实际录取分数分开；
- 区分计划数和统考名额；
- 对缺失数据降置信度；
- 高风险判断要找反方证据。

我不想把它包装成“AI 预测神器”。这类说法很有流量，但不太负责任。考研择校本来就有不确定性，工具应该做的是减少信息差，而不是制造确定感。

项目地址：

```text
https://github.com/mcxiaoxiao/kaoyan-navigator-skill
```

如果你对这个方向感兴趣，可以看看仓库结构。也欢迎贡献一些公开、脱敏的院校案例，或者帮忙完善 PDF 解析脚本。这个项目现在还很早期，但我希望它能慢慢长成一个靠谱的考研择校资料导航。
